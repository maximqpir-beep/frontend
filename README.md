# Фронтенд и бэкенд разработка — Комплексный проект

Данный репозиторий содержит 6 практических работ по дисциплине "Фронтенд и бэкенд разработка", объединенных в единый проект.  
Контрольная работа 1 представляет собой проверку работоспособности и полноты всех выполненных заданий.

---

## Структура проекта

| Папка | Название | Описание |
|-------|----------|----------|
| pr1 | CSS-препроцессоры | Карточка товара на SASS с переменными, миксинами и вложенностью |
| pr2 | Express.js API | REST API для управления товарами (CRUD) |
| pr3 | JSON и внешние API | Тестирование API в Postman, скриншоты, работа с OpenWeatherMap и ExchangeRate-API |
| pr4 | API + React | Интернет-магазин на React с подключением к бэкенду |
| pr5 | Swagger | Документация REST API с помощью Swagger UI |
| pr6 | Контрольная работа 1 | Итоговый отчет и проверка всех работ |

---

## Проверка работоспособности

### Практическая работа 1 — CSS-препроцессоры

**Задание:** Карточка товара с переменными, миксинами и вложенностью.

**Результат:**  
Файл pr1/index.html открывается в браузере, отображается стилизованная карточка товара.  
Использованы:
- переменные SASS ($primary-color, $font-stack, $border-radius)
- миксин (@mixin card-hover-effect)
- вложенная структура (BEM)

---

### Практическая работа 2 — Express.js API

**Задание:** REST API для управления товарами (CRUD).

**Результат:**  
Сервер запускается командой node app.js из папки pr2/.  
Доступны маршруты:
- GET /products — все товары
- GET /products/:id — товар по ID
- POST /products — создание товара
- PATCH /products/:id — обновление товара
- DELETE /products/:id — удаление товара

---

### Практическая работа 3 — JSON и внешние API

**Задание:** Тестирование API в Postman (не менее 5 запросов к внешнему API + тестирование своего API).

**Результат:**  
В папке pr3/screenshots/ находятся скриншоты:
- Тестирование собственного API (GET, POST, PATCH, DELETE)
- Запросы к OpenWeatherMap (погода в Москве, Лондоне и др.)
- Запросы к ExchangeRate-API (курсы валют USD, EUR)
- Запросы к JokeAPI

Подробный отчет: pr3

---

### Практическая работа 4 — API + React

**Задание:** Интернет-магазин на React с подключением к Express API.

**Результат:**  
- Сервер (pr4/server) запускается на порту 3000
- Клиент (pr4/client) запускается на порту 3001
- React-приложение получает данные с сервера и отображает карточки товаров
- Реализовано 10 товаров с полями: название, категория, описание, цена, количество на складе
- Поддерживаются операции создания, редактирования и удаления товаров

---

### Практическая работа 5 — Swagger документация

**Задание:** Добавление Swagger UI к REST API.

**Результат:**  
- Установлены пакеты swagger-jsdoc и swagger-ui-express
- В pr5/server/app.js добавлены JSDoc-аннотации для всех эндпоинтов
- Документация доступна по адресу: http://localhost:3000/api-docs
- Описаны все CRUD операции и схема User
- Поддерживается интерактивное тестирование API через Swagger UI

---

### Практическая работа 6 — Контрольная работа 1

**Задание:** Проверка работоспособности всех предыдущих работ и подготовка итогового отчета.

**Результат:**  
- Все 5 практических работ проверены и работают
- Подготовлен итоговый отчет
- Репозиторий открыт и доступен для проверки

## Как запустить проект

### 1. Клонирование репозитория

git clone 
cd project


### 2. Запуск серверной части (pr2, pr4, pr5)

Пример для pr4 (интернет-магазин):
cd pr4/server
npm install
npm run dev


### 3. Запуск клиентской части (pr4)

В отдельном терминале:
cd pr4/client
npm install
npm start


### 4. Просмотр фронтенда (pr1)

Открыть pr1/index.html в браузере.

### 5. Просмотр документации Swagger (pr5)

cd pr5/server
npm install
npm run dev

Затем открыть http://localhost:3005/api-docs
